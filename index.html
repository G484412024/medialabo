プロフェクト課題について

検索キーワードから一つを選択し、検索ボタンを押すことによって各お店のジャンルが選択したキーワードのお店の情報をネット上から取得し、ページ上に表示してくれるプログラムです。表示される情報は店名、アクセス、住所、予算、キャッチコピー、ジャンル、営業時間、最寄駅です。また、画像が表示された画像をクリックすることによってホットペッパーの該当するお店のページに移動することができます。うまく画像が表示されていないものに関しては移動することができません。

プロジェクト課題で作成したページ: project.html

数当てゲーム

ランダムに生成した数(1~10)を予想してもらうプログラムです。ユーザはテキスト欄に1~10の数字を入力し回答ボタンを押すことでユーザが入力した数が生成した数より大きい、小さい、正解のいずれかを返します。この結果を元に数字を予想して正解を当ててもらいます。回答できるチャンスは3回までです。3回入力すると正解が表示されます。4回以降は入力しても正解と比較されません。正解を当てるとそこでゲームは終了します。正解を当ててもそれ以降正解との比較は行われません。

テキスト欄には文字や0,10以上の数字を入力することは可能ですが、文字を入力すると正解との比較が正しく行われず、結果表示がうまくできません。また、1~10を超えた数字を入力すると結果は表示されますが、ランダムに生成された数字に当てはまらないのあまり有効とは言えないので、なるべく入力しない方が良いです。

数当てゲームのページ: kazuate.html

各授業回の学習事項

第1回 WWW と HTML

WebページはHTMLとCSS、JavaScript、画像・音声・動画によって構成されている。HTMLには要素と属性があり、さらに要素にはブロック要素とインライン要素がある。属性とは要素に追加する付加情報である。段落やメニューなどがブロック要素で、強調やリンクなどはインライン要素に当たる。インライン要素の中にブロック要素は配置できない。HTMLには特殊文字が5つある。

演習の成果: 演習1-4:HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

CSSによるHTMLの装飾の仕方、HTMLにどのようにCSSを追加するのか、CSSの調べ方やデバックのやり方について学んだ。さらにCSSでは、文字の大きさや色、書体、特定の要素を指定、文字の裏に背景色、画像やレイアウトなども詳細に設定できることを学んだ。

演習の成果: 演習2-3:ウェブブラウザ説明ページの装飾

第3回 JavaScript

JavaScriptはwebページ上で複雑な機能を実現するためのプログラミング言語だと理解した。JavaScriptをページに反映させる方法を学んだ。コンソールにてプログラムを確認し、VSに書き写すという手順を学んだ。オブジェクトの作り方や扱いについて学んだ。開発者ツールを使ったデバッグのやり方について学んだ。

演習の成果: 演習3-6:オブジェクトの参照

第4回 DOM と JavaScript

ボタンを押したらページが変化する処理をJavaScriptプログラムとして記述する方法を学んだ。入力欄を作るinput要素とボタン、input要素のtype属性について学んだ。要素を検索し、参照する方法を学んだ。また、ユーザが設定した値を参照し、要素のテキストを変更することによって、ユーザの設定した値で結果を表に表示させる方法について学んだ。イベントハンドラの登録と確認の仕方について学んだ。

演習の成果: 演習4-10:足し算ページ

第5回 イベント処理

DOMの要素の新規作成、要素の編集(要素の属性の追加、変更)、要素の削除の使い方について学んだ。要素の削除を用いてページ上にある表示を消すことでボタンを押すごとに前の結果を消すことができることを学んだ。画像のURLを入れることで画像も表示できることを学んだ。プログラムを工夫すると複雑な表示をさせることができることを学んだ。

演習の成果: 演習5-7:DOMプログラミング(2)

工夫点，苦心点，強調点，感想

工夫した点について

indexは学習事項をフレックスにして、文字の間隔を細かく調整しました。タイトルと本文で背景の色を変え、各項目で区切りをしっかりさせ一目でどこまでかをわかるようにしました。文字と背景の色がなるべく近くならないようにしました。タイトルを白抜きで表示させ、背景を白と黒のグラデーションで表示させましたが、少しでも見やすくなるようにタイトルの文字の後ろからも黒が滲むようにしました。

Projectではキーワードは決まっているので入力ではなく選択式の方が良いと考えました。検索結果の表示を表示させる際リスト要素を用いて何軒目の結果であるかを強調させました。偶数番目の色をかえ、行を認識しやすくしました、画像も配置して、画像をクリックすることによってページを移動できるようURLを配置しました。

苦心点について

IndexではCSSの要素の指定にとても苦労しました。背景の色をつけるのに同じ項目のp要素同士で間隔が空いてしまったりpaddingやmarginの違いがなかなか理解できなかったりしました。グラデーションを指定する際、色の範囲の指をなかなかうまくできなくて苦労しました。

Projectでは要素の配置がとても難しかったです。forを用いて情報を表示したのですが、初めは要素の指定がうまくいかず、配置する順番がぐちゃぐちゃになってしまっていました。

強調点

Indexではタイトルと各回のURLの他の部分よりも明るめの色にして目立つようにしました。その二つは一目でわかることが重要だと考え、その上授業回の別れにもなっているので、特に目立つようにしました。

Projectでは検索結果の表示で画像を表示させました。一回でなんとくでもお店の雰囲気を掴めるように画像を追加してみました。

感想

Index課題ではCSSを試行錯誤しながら色々装飾を施したおかげで要素の指定の仕方や背景の色の作り方がなんとなく理解できるようになってきました。特にグラデーションや文字に影をつける作業なんかは色々試行錯誤していてとても楽しかったです。色をつけるのは楽しかったですが、見やすいページというのはどういうものかということを考えながら作るのが少し難しかったです。JavaScriptを学んでからユーザの操作によってページが変化するというのはワクワクしました。しかし、その反面JavaScriptを理解するのは難しく、プログラムを書くのにもとても苦労しました。しかし、目で見てわかるページというのはとても作っていて楽しかったです。この授業を通して、html、CSS、JavaScriptについて色々手を動かして試行錯誤したおかげでどのように書けばどのように動くのか、なんとなく理解できたのでとても楽しく授業を受けられました。
